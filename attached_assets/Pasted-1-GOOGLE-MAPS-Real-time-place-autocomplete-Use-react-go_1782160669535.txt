Fee calculation — implement exactly this logic, step by step:

Example using ₦500 deposit:

Flutterwave fee (raw): Calculate the real FLW fee — flat ₦10.75 for amounts ≤ ₦2,500; ₦26.88 for ≤ ₦50,000
What user pays: Strip the kobo — take only the whole naira portion of the FLW fee using Math.floor(fee). User pays deposit + Math.floor(FLW_fee). Example: ₦500 + ₦10 = user pays ₦510
FLW deducts its full actual fee including kobo from the ₦510 sent — so ₦510 − ₦10.75 = ₦499.25 arrives
Admin fee: ₦20 for every ₦500 in the deposit. Formula: (deposit / 500) × 20. Examples: ₦500 → ₦20 admin, ₦1,000 → ₦40, ₦2,000 → ₦80, ₦10,000 → ₦400
Amount credited to user's account: arrived_amount − admin_fee = ₦499.25 − ₦20 = ₦479.25 credited
Both the Flutterwave fee and the admin fee are removed before crediting — the user's dashboard shows only what actually lands in their account
Show the user a full breakdown before they confirm payment:

Deposit amount: ₦500
Flutterwave fee: ₦10 (shown as whole naira only, no kobo)
Admin fee: ₦20
Total to send: ₦510

Backend — all calls to https://api.flutterwave.com/v3:
Use secret key directly as Bearer token — Authorization: Bearer ${process.env.FLW_SECRET_KEY}. No OAuth, no token exchange.

Bank transfer: POST /v3/charges?type=bank_transfer with { tx_ref, amount: String(totalToSend), email, currency: "NGN", fullname }. Account number is at response.meta.authorization.transfer_account, bank at response.meta.authorization.transfer_bank
Card: POST /v3/payments — checkout link at response.data.link
Verify bank transfer: GET /v3/transactions?tx_ref=... — confirmed when response.data[0].status === "successful"
Verify card: GET /v3/transactions/{transactionId}/verify — confirmed when response.data.status === "successful" and tx_ref matches
On confirmed payment: credit the calculated creditedAmount (not the deposit amount) to the user's balance in the database and return the new balance
